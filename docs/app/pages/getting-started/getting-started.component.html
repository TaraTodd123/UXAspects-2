UX Aspects components and charts can be used in Angular projects (Angular 6, 7, and 8 are currently supported), and those shown in the CSS section can be used in non-Angular projects.

Pre-Requisites

Instructions on this page refer to the following software:

  • Node.js (v10 or higher) — this provides the npm command, which is used to install application dependencies.
  • Git — used to download the seed project, or to build UX Aspects from source.
  • Angular CLI — used to build the seed project or create a new Angular application.
  • Grunt — used to build UX Aspects from source.

Seed Project

The easiest way to get started with UX Aspects is to download our Seed Project. This is a starter project, suitable for building a new Angular application using UX Aspects. Since it is built with Angular CLI, it is easy to extend with new pages and components.

  1. Clone the repository: git clone https://github.com/UXAspects/UXAspects-Seed.git my-project
  2. Switch to the directory which was downloaded.
  3. Install the dependencies: npm install
  4. Build and run the site using the Angular CLI development server: ng serve

A complete guide on using the Seed Project can be found here.

Add to an Existing Project

If you want to consume UX Aspects in an existing application follow the steps below:

Install UX Aspects

npm install --save @ux-aspects/ux-aspects

Install Peer Dependencies

UX Aspects has a number of peer dependencies which should be met to ensure proper functionality across all components. Refer to the output of the above command to see which dependencies are unmet, and install them manually.

Stylesheets

Add the Bootstrap and UX Aspects stylesheets to the application. If using Angular CLI, add these via the angular.json file; otherwise, they can be imported into the master stylesheet. Note that the order of import is significant.

Consuming UX Aspects Components

Import the required component module(s) into the module that will be using them. The Usage dropdown in the documentation lists the module to import, where appropriate.

Contributing

Found a bug or thought of a new feature that would benefit others? Well you can help make UX Aspects even better by contributing to the project. Contributing is easy, and all fixes are welcome, no matter how small.

A complete guide on contributing can be found here.

Building UX Aspects

  1. Clone the UX Aspects GitHub repository.
  2. Switch to the UXAspects directory.
  3. Install the dependencies: npm install
  4. Build the project: grunt build

The resources (scripts, CSS, fonts, and images) can be found in the dist folder.
